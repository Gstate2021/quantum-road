# PQC（ポスト量子暗号）学習ガイド

## 前提：数学が得意でなくても大丈夫

PQCを「完全に理解する」には格子暗号などかなり重い数学が必要。
でも必要なのはそこじゃない。

**目標は「設計・提案・判断ができる」レベル**

実装は既存ライブラリがやってくれる。
「なぜ今の暗号が量子で壊れるのか」「何に置き換えるべきか」を説明して、クライアントに移行計画を提案できること。

---

## 学ぶ順番

### STEP 1：現在の暗号の仕組みをざっくり理解する

今の暗号（RSA・AES）がなぜ成り立っているかを掴む。

- 「素因数分解が難しいから安全」くらいのレベルで十分
- YouTubeで「RSA encryption explained」で日本語動画あり

### STEP 2：NISTが標準化した4つのアルゴリズムを覚える

2024年にNISTが正式標準化した4つ。数学は不要、用途だけ知っておく。

| アルゴリズム | 用途 |
|---|---|
| ML-KEM | 鍵交換用 |
| ML-DSA | 署名用 |
| SLH-DSA | 署名用 |
| FN-DSA | 署名用 |

### STEP 3：実際にコードで動かす

数学なしでPQCアルゴリズムを体感できるライブラリ：

```bash
pip install cryptography
pip install liboqs-python  # Open Quantum Safe
```

動かして体感するのが一番早い。

## ビジネスへの落とし方

このレベルの理解があれば以下の流れで提案できる：

1. 既存システムの暗号棚卸し
2. リスク評価（RSA系を使っているか確認）
3. PQCアルゴリズムへの移行計画の提案

数学を深掘りするより、**クライアントに説明できる言葉を持つことを先にやる**のが合理的。

---

## 量子との関係性まとめ

```
量子コンピュータが実用化
      ↓
RSA・楕円曲線暗号が破られる可能性
      ↓
PQC（量子耐性アルゴリズム）に移行が必要
      ↓
その移行支援が今から実ビジネスになる
```

量子そのものを理解しなくても、**PQCは今すぐ動ける入口**として一番現実的。
